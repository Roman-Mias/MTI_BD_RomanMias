# Практика 2 — ответы на контрольные вопросы

## 1) Почему первичный ключ не может содержать NULL?
Потому что первичный ключ должен однозначно идентифицировать запись. NULL означает “нет значения”, и это нарушает уникальную идентификацию.

## 2) В чем разница между CHAR и VARCHAR?
CHAR(n) — фиксированная длина (добивает пробелами).  
VARCHAR(n) — переменная длина (хранит фактическое количество символов).

## 3) Какие типы данных вы использовали и почему?
- INT — для идентификаторов (StudentID, GroupID, SubjectID), удобно индексировать/сравнивать  
- VARCHAR — для текстовых полей (имя, фамилия, группа, факультет, email)  
- DATE — для даты рождения (без времени)
